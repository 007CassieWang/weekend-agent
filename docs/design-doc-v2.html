完整版本

本产品解决的不是"信息不够"，而是"决策太累"。从"想出去"到真正出门，中间隔着找灵感、比价格、查预约、凑人头，被切分在四个 App 里来回搬运。我们要做的，是把这条链路收进一次对话。

一、核心创新

猜问，而非读心

不预测用户想要什么，而是引导用户说清想要什么。每轮只抛一个可判断的方向——室内还是户外、带娃还是约会——用户只需点头或摇头。既避免了意图推断必然的猜错重来，也让不擅表达的用户在三轮内收敛出明确诉求。

约束前置，方案渐现

不先出方案再让用户筛选，而是先收约束再生成方案。人群、预算、档期、天气，每轮问答都在压缩候选空间。最终方案不是系统的一次性猜测，而是用户逐步勾勒的结果——可解释，可回溯。

粒度决定精度

推荐的上限取决于知识库的颗粒度。我们不以"一座商场"为推荐单元，而是下钻到有独立定价、独立时长、独立人群的子设施。标签据此分三层：属性管筛选，热度管排序，行动管执行，各司其职。

推荐即执行的起点

给出方案不是终点。行动标签直接驱动下一步：有团购则推券，需预约则代订，要凑人则建群。确认方案后，流程被推进到"可成行"，而非停在一份待办清单。

二、演进方向

👥 从单人规划到群体决策

单人跑通猜问闭环后，多人协同是顺理成章的延伸。"周末聚一下"的痛点不在没有去处，而在无人愿当组织者。猜问在群聊中同样成立：一人发起、系统抛案、各自表态、求公约数。组织者的角色被系统接管，参与者只需选择。

💬 从行程工具到社交入口

一份待确认的行程，本身就是轻量的社交货币。发起、投票、确认、同步下单——规划从个人决策变为群体共识。产品的位置，也从"查去哪"前移到"帮组局"。

🧠 从一次成行到偏好沉淀

每次选择都在补全个人与群体的口味画像。同一群人再次相约时，系统不必从零开始：成员偏好、历史选择、档期约束已经在位，方案更快、更准、更省沟通。规划的边际成本，随使用递减。
